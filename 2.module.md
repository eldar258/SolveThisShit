# Объявление и объединение констант
##### Константа
Именованное хранилище данных, которые не могут изменятся
##### Объявление константы
Как переменная, только вместо var - const
##### Объявление
Объявление переменной или константы может включать в себя идентификатор(имя), тип, начальное значение. Используется, что бы уведомить компилятор о существовании элемента.
##### Виртуальная память
механизм, который позволяет программам использовать больше памяти, чем доступно оперативной памяти. Виртуальная память разбивается на сегменты:
- `Сегмент кода` - содержит исполняемый код программы.
- `Сегмент данных` - содержит данные программы, такие как переменные.
- `Сегмент стека` - содержит временные данные, такие как локальные переменные и адреса возврата.
##### Группа констант
набор связанных констант, которые объявляются вместе в одном блоке const
##### Атрибуты памяти
чтение и/или запись
##### iota
идентификатор, который используется для создания последовательности чисел в группе констант. При каждом вызове значение увеличивается на 1 начинаясь с 0.
# Типизированные и нетипизированные константы
Константы которые имеют или не имеют определенного типа данных
##### Нетипизированные
Используются в выражениях, которые ожидают разные типы. Golang автоматически преобразует значение константы в нужный тип.
##### **Строгая типизация** 
свойство языка программирования, которое требует, чтобы переменные имели определенный тип данных, и не позволяет использовать переменные с другим типом данных
##### Тип данных 
свойство переменной, которое определяет, какие значения могут быть присвоены этой переменной и какие операции с ней могут быть выполнены.
##### Целочисленный тип данных
тип данных, который может хранить только целые числа
##### Какие есть целочисленные типы данных
`int8, int16, int32, int64, uint8, uint16, uint32, uint64, int, uint, rune, byte`
##### **Числа с плавающей точкой**
числа, которые могут иметь дробную часть и представлены в виде мантиссы, порядка(экспонента, показатель) и знака числа
##### **Мантисса** 
дробная часть числа с плавающей точкой.
##### **Порядок** 
целочисленная часть числа с плавающей точкой, которая определяет масштаб числа.
##### float на низком уровне
| Тип | Знак | Мантисса | Порядок | Всего бит |
| --- | --- | --- | --- | --- |
| float32 | 1 | 23 | 8 | 32 |
| float64 | 1 | 52 | 11 | 64 |
##### IEEE 754
стандарт, который определяет формат представления чисел с плавающей точкой в компьютерах
##### Применение bitwise к числам с плавающей точкой
функции math.Float32bits и math.Float64bits преобразуют числа с плавающей точкой в их двоичное представление. math.Float32frombits или math.Float64frombits преобразует число обратно
##### unsafe.Sizeof
которая возвращает размер типа в байтах
# Использование сигнатур функций
##### Функция
блок кода, который выполняет определенную задачу и к которому можно обратиться
##### Сигнатура чего либо
общая информация о чем либо
##### **Сигнатура функции**
определяет общую информацию о функции, такую ​​как имя (или его отсутствие), типы параметров, тип возвращаемых значений. Является частью объявления ф-ии
##### Тело функции 
блок кода, который выполняется при вызове функции. Является частью реализации ф-ии
# Применение аргументов функций
##### Аргумент функции
значение, переданное при вызове функции
##### Результат функции
значение, возвращенное функцией (после выполнения).
# Использование вариативных функций
##### Вариативная функция
функция, которая может принимать переменное количество аргументов. В Golang у сигнатуры такой ф-ии у последнего параметра стоит многоточие `...` перед типом
##### Примеры вариативных ф-ий из стандартной библиотеки
`fmt.Printf(), append()...`
# Возвращение значений функций
##### Множественное возвращение значений 
возможность функции возвращать более одного значения
# Обработка ошибок функций
##### **Error return** 
механизм возврата ошибок из функции в Golang. Функция может вернуть значение ошибки, которое может быть обработано в вызывающей функции, или значение nil, если ошибка не произошла
##### error
интерфейс ошибки в Golang
##### Как создать ошибку
`fmt.Errorf("error text")` или `errors.New("error text")`
##### Обработка ошибок
процесс обнаружения и устранения ошибок
##### Как обработать ошибку
```Go
value, err := f("some argument")
if err != nil {
	//обработка ошибки
}
```
##### panic
механизм аварийной остановки программы в Golang.
##### «Чистые функции»
функции, которые не имеют побочных эффектов и всегда возвращают одинаковый результат для одинаковых аргументов
# Применение анонимных функций
##### Анонимная функция 
функция, которая не имеет имени и определяется внутри другой функции
##### Лямбда функция
анонимная функция, которая может быть передана в качестве аргумента функции или значения переменной
# Использование рекурсий и замыкания
##### Рекурсия
процесс, при котором функция вызывает сама себя.
##### Базовый случай
случай, когда функция не вызывает сама себя, а возвращает результат
##### Рекурсивный случай
случай, когда функция вызывает сама себя с измененными параметрами
##### Замыкание (closure)
получение внутренней функцией доступ к переменным из внешней области видимости.
##### Внешняя область видимости
область видимости, в которой были созданы переменные, на которые ссылается замыкание
# Юнит-тестирование
##### Юнит-тестирование
процесс тестирования отдельных блоков кода (функций), для проверки их корректности и соответствия требованиям.
##### Тестовая функция
функция, которая проверяет корректность работы другой функции.
##### Пример тестовой ф-ии
```Go
func TestFunc(t *testing.T) { 
	t.Errorf("")
}
```
#### Табличное тестирование
тестирования, при котором тестовые данные и ожидаемые результаты хранятся в таблице
# Написание бенчмарка
##### Бенчмарк в программировании
процесс измерения производительности программного обеспечения
##### Пример функции бенчмарка 
```Go
func BenchmarkMyFunction(b *testing.B) {
	for i := 0; i < b.N; i++ {
		myFunction() 
	} 
}
```
##### Reset timer в бенчмарке
функция, которая позволяет сбросить таймер перед выполнением каждой итерации бенчмарка.
# Оптимизация с помощью бенчмарка
##### Оптимизации кода с помощью бенчмарка
1. Написать код, который нужно оптимизировать.
2. Написать бенчмарк для измерения производительности кода.
3. Запустить бенчмарк и проанализировать результаты.
4. Оптимизировать код на основе результатов бенчмарка.
5. Повторить шаги 2-4 до тех пор, пока производительность не будет удовлетворительной.
